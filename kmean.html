Project

While working on UK retail data, I realized that standard RFM features didnâ€™t fully capture repeat behavior. To address this, I researched and implemented an additional metric: Interpurchase Time, improving the behavioral clarity of clusters beyond frequency counts.

Methodology

  • Data Preparation: Cleaned 541,909 transactions; removed missing CustomerIDs and outliers.
  • Feature Engineering: Calculated RFM + Interpurchase Time + Impulse Score (proportion of 2-day repurchases).
  • Clustering: Applied KMeans (k=5, Elbow & Silhouette selected); visualized behavior by segment.
  • Emotion Analysis: Simulated user feedback per segment and ran sentiment analysis using VADER to quantify tone.

Key Outcomes

  • Identified 5 behaviorally distinct clusters using quantitative and emotional features.
  • Mapped personalized UX strategies to each cluster (e.g. urgency prompts, loyalty perks, comeback offers).
  • Built a real-time classifier and deployed a Streamlit tool to predict customer segments from raw behavior input.

Strategic Summary

This project demonstrates a full-cycle customer segmentation pipeline:

  • Clustering: Data-driven segmentation using time-sensitive behavioral indicators.
  • Interpretation: Emotion simulation enabled a deeper understanding of customer intent and tone.
  • UX Planning: Segment-level messaging strategies designed to match behavioral traits.
  • Operationalization: A real-time model now maps users to clusters for dynamic personalization.

The output is a deployable framework for targeting, not just analysis.
