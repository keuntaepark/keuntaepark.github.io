Removing Returns (InvoiceNo starts with 'C')¶

In this dataset, rows with InvoiceNo starting with 'C' indicate cancellations or returns, according to the UCI Online Retail dataset documentation.

These entries often include:

  • Negative Quantity

Such records reflect reversed transactions and can distort behavior-based features like frequency, monetary value, and impulse patterns.

To ensure clean, intent-driven purchase data, we exclude them from analysis:

  • December shows a sudden drop, which aligns with the fact that the dataset ends on December 9, 2011 — not a full month.
  • Mid-year months (June–August) show relatively flat performance, possibly reflecting seasonal consumer behavior.

Insights

This time-based trend significantly affects features like Recency and Monetary, particularly for customers active during the holiday season (November–December).

To account for this seasonality, we apply feature normalization prior to clustering, ensuring that temporary spikes or cut-off effects from incomplete months do not distort the segmentation outcomes.

Features for Clustering¶

  • Recency - Days since last purchase → captures customer freshness
  • Frequency – Total transactions → reflects engagement & loyalty
  • Monetary – Total spending → measures customer value
  • InterpurchaseTime – Avg. days between purchases → buying rhythm
  • ImpulseScore – % of purchases ≤ 2 days apart → impulsiveness indicator

Together, these features form a behavior-rich profile of each customer, enabling more nuanced segmentation than RFM alone.

Frequency, Monetary, InterpurchaseTime

These features contain extreme outliers.
A few customers purchased hundreds of times or spent tens of thousands, while most others remained in a much lower range.

Standardization alone cannot resolve this imbalance—
the presence of large values inflates the mean and standard deviation, weakening comparisons between typical customers.

To address this, a log transformation (log1p) is applied first.
This reduces the impact of large values and compresses the scale, creating a more balanced input for clustering.

In [17]:
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt
from sklearn.metrics import silhouette_score

log_cols = ['Frequency', 'Monetary', 'InterpurchaseTime']
for col in log_cols:
    rfm[col] = np.log1p(rfm[col])

scaler = StandardScaler()
X_scaled = scaler.fit_transform(rfm[features])

inertia = []
K_range = range(2, 10)

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init='auto')
    kmeans.fit(X_scaled)
    inertia.append(kmeans.inertia_)

plt.figure(figsize=(6, 4))
plt.plot(K_range, inertia, marker='o')
plt.title('Elbow Method for Optimal K')
plt.xlabel('Number of Clusters (K)')
plt.ylabel('Inertia')
plt.grid(True)
plt.show()

Based on the Elbow Method and Silhouette Scores, K=4 and K=5 were selected for comparison:

  • K=4 is the elbow point where the rate of inertia reduction slows significantly.
  • K=5 yields the highest silhouette score (0.3738), suggesting better separation between clusters.

Both values are reasonable. Clustering will be performed for K=4 and K=5, followed by a behavioral interpretation of each segment.

Monetary Total amount spent High-value customer Low spender
InterpurchaseTime Avg. days between purchases Irregular / slow cycle Consistent / clustered buying
ImpulseScore % of purchases within 2 days of the last one Impulsive / reactive behavior Deliberate / planned behavior

While K=4 provides a solid segmentation structure, K=5 introduces a valuable refinement:

  • In K=4, Cluster 0 grouped all low-frequency impulsive buyers together.
  • In K=5, that group is further split into two distinct profiles:
    • One with moderate recency (Cluster 0)
    • One with very high recency (Cluster 4)

This distinction allows us to separate active impulse buyers from lapsed one-time responders

Transition to Emotion-Driven Enhancement¶

Behavioral clusters show what users do.
To go further, we need to understand how they might feel.

Next, we simulate customer feedback and apply sentiment analysis.
This links each cluster to emotional tone—adding depth for better messaging and UX decisions.

from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
import pandas as pd

synthetic_reviews = {
    0: "I usually shop when there’s a big sale, and I buy quickly without thinking much.",
    1: "I browse sometimes and buy if I find something interesting.",
    2: "I always come back. I love new arrivals and enjoy spending here.",
    3: "I shopped once, it was okay, but I don’t feel the need to return.",
    4: "It's been a while. I bought on impulse before, but haven’t looked again."
}

analyzer = SentimentIntensityAnalyzer()
sentiment_results = {
    cluster: analyzer.polarity_scores(text) for cluster, text in synthetic_reviews.items()
}

emotion_df = pd.DataFrame(sentiment_results).T
emotion_df.index.name = 'Cluster'
emotion_df = emotion_df[['pos', 'neu', 'neg', 'compound']]
emotion_df.round(2)
