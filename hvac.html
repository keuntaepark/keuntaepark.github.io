Project Overview

  • Duration: Jan 29 – Apr 30, 2023
  • Team: 4BY4 designer, Unity developer, external partners
  • Role: Lead, Data Analysis & Project Management

Problem

In this client project to visualize HVAC systems in Unity, we identified performance bottlenecks caused by overloaded high-resolution assets. There was a lack of behavioral data, making UX direction unclear. Additionally, we needed to reconcile conflicting expectations between design (visual fidelity) and engineering (performance and stability).

Action

  • Bridge between Client & Technical Teams: Quickly learned HVAC system structure and industry context; translated requirements into technical specs and relayed them effectively to the Unity team
  • Behavioral Data Analysis: Used Python to generate heatmaps of user clicks and dwell time → lowered texture resolution in underused 3D areas to improve loading speed
  • Team Alignment & Transparency: Created a real-time Miro dashboard to share feedback, issue tracking, and progress with all stakeholders
  • Data-informed Design Trade-offs: Balanced design aesthetics and system performance through structured analysis → optimized Unity scene configuration

Result

  • +100% visitor growth: Improved visibility and sales lead generation
  • +15% session duration: Higher engagement and better UX
  • 20% faster page load time: Reduced bounce rates and improved satisfaction

Learning & Growth

This project taught me how critical structured data is in resolving cross-functional conflicts. I shifted from intuition-driven decisions to evidence-based alignment—now, I prioritize early-stage measurement strategies in every project.

Follow-up Strategy

  • Proposed integration with Google Analytics and A/B testing to analyze behavior by user segment
  • Suggested connecting UX improvements to conversion flows to support sales outcomes

프로젝트 개요

  • 프로젝트 기간: 2023.01.29 – 2023.04.30

문제 정의 (Problem)

HVAC 시스템을 Unity로 시각화하는 클라이언트 프로젝트에서, 고해상도 자산 사용으로 인한 퍼포먼스 병목이 발생했고, 사용자 행동 데이터 부재로 UX 개선의 방향성이 불분명했습니다. 또한 클라이언트 요구(디자인 퀄리티)와 개발팀 요구(속도 및 안정성)가 충돌하며 프로젝트 정렬에 어려움이 있었습니다.

나의 기여 (Action)

  • 클라이언트–작업자 간 브릿지 역할 수행: HVAC 구조와 산업 컨텍스트를 빠르게 학습한 뒤, 기술 사양을 개발팀에 번역하고 사용자 관점에서 요구사항 정리
  • 행동 데이터 기반 분석: Python을 활용해 3D 공간별 체류시간, 클릭 빈도 heatmap 분석 → 저활성 영역에 낮은 해상도 적용하여 속도 최적화
  • 정렬 구조 설계: Miro 기반 대시보드를 구축해 클라이언트 피드백, 내부 진척도, 병목 이슈를 시각화 → 모든 이해관계자 간 투명성 확보
  • 기술 설계 조율: 데이터 기반 설계를 통해 디자인 품질과 성능 간 trade-off 해결 → Unity 최적화 방향 도출

성과 및 임팩트 (Result)

  • 방문자 수 +100% 증가: 가시성 향상 및 리드 생성에 기여
  • 세션 시간 +15%: UX 개선 효과와 사용자 몰입도 증대
  • 페이지 로딩 속도 20% 개선: 초기 진입 이탈률 감소

학습 및 성장

기술·디자인·비즈니스 간 갈등 상황에서, 단순한 감이 아닌 데이터를 통해 구조적으로 문제를 조율하는 경험을 얻었습니다. 이후 프로젝트에서 항상 데이터 수집 및 해석 단계를 먼저 고려하는 습관이 생겼습니다.

후속 제안 및 확장성

  • Google Analytics 연계 및 A/B 테스트를 통한 사용자 세그먼트별 행동 분석
  • UX 개선 → 컨버전 → 영업전환까지 연결하는 데이터 기반 흐름 설계 제안
