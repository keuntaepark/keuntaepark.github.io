Project Overview

As project manager and data lead, I drove the LG Electronics HVAC Virtual Experience (Jan–Apr 2023) by aligning design, engineering, and external partners. The goal: deliver a high-performance virtual showroom for B2B engagement, supported by data-driven UX improvements and cross-functional alignment.

Key Challenges

  • Identified heavy Unity assets as the main bottleneck using walkthrough tests, developer logs, and user complaints.
  • Discovered missing user behavior data, making UX improvement decisions speculative.
  • Mediated design vs. engineering conflicts over visual fidelity vs. performance priorities.

Approach & Strategy

  • Led heatmap analysis in Python to detect low-traffic 3D zones; recommended asset downgrades to improve load speed.
  • Acted as cross-team facilitator, ensuring clear priorities and translating client goals into technical roadmaps.
  • Built a live Miro dashboard to track feedback, align progress, and keep stakeholders synchronized.

Key Outcomes & Business Impact

  • +100% YoY visitor growth, contributing to increased B2B leads and stronger sales pipeline.
  • +15% longer session times, reflecting better user engagement and navigation.
  • 20% faster load times, boosting user retention and experience quality.

What I Learned

Having faced the limitations of intuition-based decision-making, I realized the importance of data-driven approaches for solving problems and aligning teams. Since then, I’ve made it a practice to embed analysis and measurement frameworks early in projects.

Client Recommendations & Future Plans

Post-project, I proposed Google Analytics integration and A/B testing to segment user behavior and directly link UX optimizations to business outcomes, such as lead conversions and retention improvements.

프로젝트 개요

프로젝트 기간: 2023.01.29 – 2023.04.30
참여 인원: 4BY4 디자이너, Unity 개발자, 외부 협력사
역할: 데이터 분석 및 프로젝트 관리 리드

핵심 문제 정의

  • 클라이언트 인터뷰 및 로그 분석으로 Unity 자산 과중이 주요 성능 병목임을 파악
  • 사용자 행동 데이터 부재로 UX 개선 방향성이 불명확함을 확인
  • 디자인팀의 시각 품질 요구와 엔지니어링팀의 퍼포먼스 요구 간 갈등 조율 필요

접근 방식 및 전략

  • Python으로 Heatmap 클릭·체류 데이터 분석 → 저활성 3D 구간 화질 낮춤 → 속도 최적화
  • Miro 대시보드를 구축해 피드백 및 진행 현황 실시간 공유, 팀 간 정렬 강화
  • 클라이언트 목표를 기술 설계로 전환, 엔지니어링팀과 협력해 Unity 최적화 설계

주요 성과 및 비즈니스 임팩트

  • 방문자 수 전년 대비 +100% → 신규 리드 창출 및 영업팀 연결 기회 증가
  • 세션 지속시간 +15% → 사용자 참여 및 UX 개선 효과 반영
  • 페이지 로딩 속도 20% 개선 → 사용자 이탈률 감소 및 만족도 증가

주요 학습 및 성장

감과 직관에 의존한 논의의 한계를 직접 겪으며, 데이터 기반 접근이 문제 해결과 협업 모두에서 필수적임을 깨달음. 이로 인해 이후 프로젝트에서는 초기부터 분석과 측정 체계를 고려하는 습관을 기르게 됨.

클라이언트 제안 및 다음 단계

Google Analytics 연계 및 A/B 테스트로 사용자 세그먼트별 행동을 분석하고, UX 개선을 매출 전환까지 연결할 방안을 탐색할 것을 제안.
