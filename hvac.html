Problem

High-resolution assets caused **performance bottlenecks (slow loads, drop-offs)** in our Unity-based HVAC visualization. Lack of behavioral data blurred UX direction. Critically, **design (fidelity) and engineering (performance) expectations clashed**, hindering project progress.

Action

  • Product & Technical Bridging: Translated client requirements into technical specs for Unity dev team, ensuring clear communication.
  • Behavioral Data Analysis for Optimization: Led Python-based analysis (heatmaps of clicks/dwell time) to strategically lower texture resolution in underused 3D areas, improving loading speed.
  • Stakeholder Alignment & Transparency: Created a real-time Miro dashboard to share feedback, issues, and progress, ensuring transparency and team alignment.
  • Data-Informed Design Trade-offs: Balanced design aesthetics and system performance using structured data analysis as objective evidence, optimizing Unity scene configuration.

Result

  • Increased User Engagement & Lead Generation: Achieved +100% visitor growth and +15% session duration, indicating higher engagement and better UX.
  • Improved Technical Performance: Reduced page load time by 20%, significantly decreasing bounce rates and improving user satisfaction.
  • Enhanced Cross-Functional Collaboration: Successfully reconciled conflicting stakeholder expectations using data, leading to efficient and harmonized project execution.

PM Learnings & Future Vision

This project was pivotal, teaching me the critical role of structured data in resolving complex cross-functional conflicts. I shifted from intuition-driven to evidence-based alignment, now prioritizing early-stage measurement strategies.

  • Proposed Google Analytics & A/B testing integration for user segment behavior analysis and personalized UX optimization.
  • Suggested linking UX improvements to conversion flows and sales outcomes for measurable business value.

문제 정의 (Problem)

Unity 기반 HVAC 시각화 프로젝트에서, 고해상도 자산으로 인한 **퍼포먼스 병목 (느린 로딩, 사용자 이탈)** 발생. 행동 데이터 부재로 UX 방향이 불분명했습니다. 결정적으로 **디자인 (퀄리티)과 개발 (퍼포먼스) 간 기대치 충돌**이 프로젝트 진행을 저해했습니다.

나의 기여 (Action)

  • 제품 및 기술 브릿지 역할 수행: 클라이언트 요구사항을 Unity 개발팀에 기술 사양으로 번역, 명확한 소통 확보.
  • 최적화를 위한 행동 데이터 분석: Python 기반 분석(클릭/체류 시간 히트맵)을 주도, 저활성 영역 텍스처 해상도를 낮춰 로딩 속도 개선.
  • 이해관계자 정렬 및 투명성 확보: Miro 대시보드를 구축, 피드백/이슈/진척도를 공유하여 투명성과 팀 정렬을 확보.
  • 데이터 기반 디자인 트레이드오프: 구조화된 데이터 분석을 객관적 증거로 활용, 디자인과 퍼포먼스 균형을 조율하여 Unity 씬 구성을 최적화.

성과 및 임팩트 (Result)

  • 사용자 참여 및 리드 생성 증대: 방문자 수 +100% 증가, 세션 시간 +15% 증대, 높은 참여도 및 UX 개선 효과 입증.
  • 기술적 퍼포먼스 향상: 페이지 로딩 속도 20% 단축, 이탈률 감소 및 사용자 만족도 향상.
  • 교차 기능 협업 강화: 데이터 기반으로 상충되는 이해관계자 기대치를 성공적으로 조정, 효율적이고 조화로운 프로젝트 실행.

PM 학습 및 미래 비전

이 프로젝트는 복잡한 교차 기능 갈등 상황에서 구조화된 데이터의 중요성을 깨닫게 한 경험입니다. 직관에서 증거 기반 정렬로 전환하며, 초기 단계 측정 전략을 우선시하는 철학을 갖게 되었습니다.

  • Google Analytics 및 A/B 테스트 연동 제안, 사용자 세그먼트별 행동 분석 및 개인화된 UX 최적화 모색.
  • UX 개선을 전환율 및 매출 성과와 연결하는 데이터 기반 흐름 설계 제안.
