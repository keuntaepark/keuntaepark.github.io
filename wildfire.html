Problem

Conventional wildfire forecasts tend to rely on qualitative judgments or single-direction outputs, which are insufficient for rapid and spatially informed decisions in emergency response. Wildfires often spread in multiple directions depending on wind, terrain, and other variables, but few systems quantify this complexity in a usable format for field teams.

Action

  • Reframed the Prediction Target: Modeled wildfire spread as 8-directional probability vectors to reflect multidirectional behavior and simplify map-based visualization (Leaflet.js)
  • Model Benchmarking: Compared XGBoost, LGBM, and RandomForest; selected RandomForest for its higher cosine similarity (avg. 0.58) and stable performance
  • Data Pipeline Engineering: Automated time-aligned and geospatial alignment of ERA5 wind data (NetCDF) with wildfire incident data
  • Prediction Reliability Boost: Combined real-time wind API forecasts with historical model predictions using weighted average (Î±=0.7)
  • Collaboration-Ready Output: Standardized prediction output format as `predicted_spread.json` to ensure integration readiness with frontend and external teams

Result

  • Field-Ready UX: Interactive 8-directional spread map with click-based inputs â†’ intuitive for non-technical users
  • Scalable Architecture: Decoupled API/model/UI layers â†’ extendable to terrain, vegetation, or emergency alert systems
  • Decision Support: Visual predictions enable proactive planning and earlier evacuation triggers
  • Model Interpretability: Used SHAP to build an explainable ML workflow for transparency and ongoing model refinement

Business/Public Sector Relevance

  • Designed for adoption by wildfire response agencies with fine-tuned data granularity and responsive UX
  • Planned integration with Korea Meteorological Administration and Forestry APIs for field deployment
  • Scalable as a public safety tool or climate-tech startup foundation
