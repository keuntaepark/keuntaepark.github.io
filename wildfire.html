Wildfire Spread Predictor

← Back to Portfolio

Problem

Conventional wildfire forecasts are critically flawed as they fail to quantify uncertainty, preventing decision-makers from gauging prediction reliability. They also fail to dynamically fuse stable historical models with volatile real-time data, or model how the fire's scale changes with current conditions. This results in an incomplete and untrustworthy picture for field teams.

Action

  • Redesigned the Prediction Target: Redefined the prediction goal from simple probabilities to an 8-directional vector of (probability, uncertainty) pairs, quantitatively presenting the confidence of each forecast.
  • Implemented Dynamic Alpha Algorithm: Discarded a fixed-weight approach. Introduced `calculateDynamicAlpha`, a function that calculates a dynamic weight based on real-time wind speed and model uncertainty, automatically balancing trust between historical and real-time data.
  • Context-Aware Spread Modeling: Developed `calculateDynamicMaxD` to dynamically compute the maximum spread distance (MaxD) based on real-time wind speed and humidity, enabling more realistic simulations.
  • Advanced Data Processing: Implemented an enhanced interpolation model (`interpolateProb`) that calculates a final uncertainty score for any arbitrary point by assessing the uncertainty of the four nearest data points.
  • Architected for Collaboration: Standardized all prediction outputs to the `predicted_spread.json` format and decoupled the API/Model/UI architecture to improve team collaboration and maintenance efficiency.

Result

  • Confidence-Based Decision Support: Provided uncertainty metrics (±%) with every prediction, empowering commanders to make more informed and responsible decisions based on the data's confidence level.
  • Improved Situational Accuracy: The dynamic weighting and spread distance calculations lead to predictions that react more accurately to changing weather conditions, such as wind gusts or dry spells.
  • Nationwide Predictive Coverage: The advanced interpolation model reliably provides predictions with quantifiable confidence levels even for areas without training data.
  • Secured Scalability & Improvement Path: The decoupled architecture facilitates easy integration of future variables (e.g., terrain), while a SHAP-based analysis provides a clear routine for continuous model improvement.

Business/Public Sector Relevance

  • Designed for adoption by wildfire response agencies with fine-tuned data granularity and responsive UX.
  • Planned integration with Korea Meteorological Administration and Forestry APIs for field deployment.
  • Scalable as a public safety tool or climate-tech startup foundation.

문제 정의 (Problem)

기존 산불 예보는 예측의 신뢰도를 나타내는 불확실성(Uncertainty) 정보를 제공하지 않으며, 안정적인 과거 데이터와 변동성이 큰 실시간 기상 정보의 동적 융합에 실패합니다. 또한 기상 조건에 따라 변화하는 확산 규모를 반영하지 못해, 현장 지휘관에게 불완전하고 신뢰하기 어려운 정보를 제공합니다.

나의 기여 (Action)

  • 예측 목표 재설계: 단순 확률이 아닌 ‘(확률, 불확실성)’ 쌍으로 구성된 8방향 벡터를 예측 목표로 재설계하여, 예측 결과의 신뢰도를 정량적으로 제시.
  • 동적 가중치(Dynamic Alpha) 알고리즘 도입: 고정 가중치 방식을 폐기. 실시간 풍속과 모델 불확실성을 변수로 동적 가중치를 계산(`calculateDynamicAlpha`)하여, 상황에 따라 과거 모델과 실시간 데이터의 신뢰도를 자동 조절.
  • 상황인지형 확산 모델링: 실시간 풍속과 습도를 기반으로 최대 확산 거리(MaxD)를 동적으로 계산(`calculateDynamicMaxD`)하여, 기상 조건에 따른 확산 규모를 현실적으로 시뮬레이션.
  • 데이터 처리 고도화: 임의 지점 예측 시, 4개의 인접 데이터 불확실성을 종합하여 최종 불확실성을 도출하는 개선된 보간(Interpolation) 모델(`interpolateProb`)을 적용.
  • 협업용 구조화: 모든 예측 결과를 `predicted_spread.json` 포맷으로 표준화하고, API/모델/UI 아키텍처를 분리하여 팀 간 협업 및 유지보수 효율성 확보.

성과 및 시사점 (Result)

  • 신뢰도 기반 의사결정 지원: 모든 예측에 불확실성(±%) 정보를 함께 제공하여, 현장 지휘관이 데이터의 신뢰 수준을 인지하고 더 책임 있는 판단을 내리도록 지원.
  • 상황 대응형 예측 정확도 향상: 동적 가중치와 확산 거리 계산을 통해, 돌풍이나 건조 특보 등 급변하는 기상 상황에 더 정확하게 반응하는 예측 결과 도출.
  • 전국 단위 예측 커버리지 확보: 고도화된 보간 모델을 통해, 학습 데이터가 없는 지역에서도 정량적 신뢰도를 갖춘 예측 정보를 안정적으로 제공.
  • 실무 확장성 및 개선 루틴 마련: 분리된 아키텍처로 지형 등 추가 변수 연동이 용이하며, SHAP 기반 분석을 통해 지속적인 모델 개선 루틴 확보.

비즈니스/공공 확장성

  • 산림청, 소방청 등 재난 대응 기관 맞춤형 UI 및 데이터 granularity 설계 중.
  • 기상청 실시간 API 연동 기반 실사용자 대상 베타 테스트 예정.
  • 공공 정책 및 기후 기술 스타트업으로의 스핀오프 가능성 검토.

Development Process

ver.01: Core Concept Validation

Problem: Text-based forecasts lacked spatial context.
Action: Built an MVP with a simple fan-chart to test the core hypothesis that map-based visuals would be more effective for field decisions.

Change: Text-based report → Map-based visual forecast.

Tech: Vanilla JS, Leaflet.js

핵심 가설 검증

문제: 텍스트 예보는 공간적 맥락이 부족.
해결: 지도 기반 시각화가 더 효과적일 것이라는 핵심 가설을 검증하기 위해, 간단한 부채꼴 차트 형태의 MVP를 구축.

ver.02: Enhancing Visual Realism

Problem: The initial fan-chart was too abstract for precise analysis.
Action: Replaced it with a grid-based heatmap using interpolation, significantly improving visual realism and optimizing the UX for mobile.

Change: Abstract fan-chart → Granular grid heatmap.

Tech: Client-side Interpolation, Responsive CSS

시각적 현실감 강화

문제: 초기 부채꼴 차트는 정밀한 분석에 한계.
해결: 시각적 현실감을 높이기 위해 보간법을 적용한 그리드 기반 히트맵으로 교체하고, 모바일 UX를 최적화.

ver.03: Upgrading the Prediction Engine

Problem: A static model could not adapt to changing weather.
Action: Upgraded the prediction engine with a dynamic weighting system, achieving a stable cosine similarity of 0.5875 with the RandomForest model.

Change: Static prediction model → Adaptive engine with dynamic weighting.

Tech: Python, Scikit-learn (RandomForest), SHAP

예측 엔진의 지능화

문제: 고정된 모델은 변화하는 기상에 적응 불가.
해결: RandomForest 모델로 안정적인 코사인 유사도 0.5875를 달성하는 동적 가중치 시스템을 도입하여, 상황 적응적 정확도를 향상.

ver.04: Ensuring Operational Reliability

Problem: Disjointed information and UI bugs were hindering quick decision-making.
Action: Overhauled the UI, integrating all data into a stable, single-view dashboard and eliminating 100% of critical layout bugs.

Change: Disjointed UI → Stable, single-view dashboard.

Tech: Flexbox, Advanced CSS, DOM Manipulation

운영 신뢰성 확보

문제: 분산된 정보와 UI 버그가 빠른 의사결정을 저해.
해결: 모든 데이터를 안정적인 단일 대시보드에 통합하는 UI 개편을 통해 치명적 레이아웃 버그를 100% 해결하고 운영 신뢰성을 확보.

ver.05: Providing Decision Confidence

Problem: Users needed to know not just the prediction, but how much to trust it.
Action: Introduced a visualized uncertainty score (±%) for each of the 8 directions, evolving the tool into a quantifiable decision-support instrument.

Change: Forecast-only output → Forecast + Quantified Uncertainty.

Tech: Client-side Dynamic Calculations

의사결정의 확신 제공

문제: 사용자는 예측뿐 아니라 그 신뢰도까지 파악해야 할 필요.
해결: 8개 방향별 불확실성(±%) 점수를 시각화하여, 예보 도구를 정량적 의사결정 지원 도구로 발전시킴.
