Wildfire Spread Predictor

← Back to Portfolio

Problem

Existing forecast systems are rendered ineffective during the critical "Golden Time" of a wildfire response. Obsessed with achieving a perfect prediction by modeling all variables, they are too slow to be actionable when it matters most. This creates a paradox: the pursuit of a perfect forecast prevents a fast, effective response.

Action

  • Product Strategy Pivot: From 'Perfect Prediction' to 'Rapid Response Support'
    Established 'time-to-forecast' as the primary success metric, shifting the team's focus from absolute accuracy to speed and actionability.
  • Lightweight, Prioritized Modeling
    Designed a lightweight model that prioritizes the most influential real-time variables (e.g., wind, humidity) to maximize prediction speed. For all other factors, the model selectively fuses pre-trained ML outputs, striking a strategic balance between speed and comprehensive accuracy.
  • Introducing a Confidence Layer to Address the Trade-off
    To compensate for the deliberate speed-for-accuracy trade-off, every forecast includes a quantitative 'Uncertainty' score (±%). This builds user trust and enables informed risk assessment.
  • Architecture for Agility
    Built a modular API/Model/UI architecture to enable rapid feature deployment and iterative improvement in a high-stakes environment.

Result

  • Secured the "Golden Time" for Initial Response
    Reduced forecast generation time from hours to minutes, securing the critical window for commanders to formulate initial suppression strategies.
  • Enabled Confident, High-Velocity Decision-Making
    Empowered field commanders to make rapid yet responsible decisions by providing a clear, quantitative measure of the forecast's confidence level.
  • Achieved High Situational Accuracy
    The model responds more accurately to sudden environmental changes (e.g., wind gusts) by assigning greater weight to real-time variables.
  • Built a Foundation for Scalable Growth
    The modular architecture is prepared for the phased integration of new variables (e.g., terrain, vegetation) and APIs from public agencies (e.g., meteorological and forestry services).

문제 정의 (Problem)

산불 대응의 성패를 가르는 '골든 타임'에 기존 예보 시스템은 사실상 무력합니다. 모든 변수를 고려한 완벽한 예측에 집착한 나머지, 예측 시간이 과도하게 소요되어 가장 중요한 초기 대응 시점을 놓치기 때문입니다. 즉, '느리고 완벽한 예측'이 '빠르고 효율적인 대응'을 방해하는 역설이 발생합니다.

나의 기여 (Action)

  • 제품 전략 재설계: '완벽한 예측'에서 '골든 타임 내 신속 대응 지원'으로 목표 전환
    '정확도'의 기준을 재정의하고, '예측 소요 시간 단축'을 최우선 목표(Key Metric)로 설정.
  • 선택과 집중을 통한 경량화 모델 설계
    예측 속도 극대화를 위해, 확산에 가장 큰 영향을 주는 실시간 변수(바람, 습도)를 우선 분석하는 모델 채택. 그 외 변수들은 사전 학습된 ML 모델 결과를 선택적으로 융합하여, 속도와 정확도의 현실적인 균형점 확보.
  • '빠른 예측'의 신뢰도 보완 장치 도입
    속도를 위해 일부 포기한 완벽성을 보완하고, 사용자의 신뢰를 얻기 위해 모든 예측 결과에 '불확실성(Uncertainty, ±%)' 지표를 함께 제공.
  • 신속한 배포 및 반복 개선을 위한 구조 설계
    빠른 기능 추가 및 안정적인 실시간 데이터 처리를 위해, API/모델/UI 아키텍처를 모듈형으로 분리하여 구축.

성과 및 시사점 (Result)

  • '골든 타임' 확보 및 초기 대응 능력 강화
    예측 소요 시간을 시간 단위에서 분 단위로 획기적으로 단축, 현장 지휘관의 초기 진화 전략 수립 시간 확보.
  • 신뢰도 기반의 신속한 의사결정 환경 구축
    불확실성 지표를 통해, 빠르지만 불완전할 수 있는 데이터의 신뢰 수준을 지휘관이 직접 인지하고 최적의 초기 대응 전략 수립 가능.
  • 상황 대응형 예측 정확도 확보
    핵심 변수에 대한 실시간 가중치 부여로, 돌풍 등 급변하는 기상 상황에 더 정확하게 반응하는 예측 모델 구현.
  • 지속 가능한 확장 기반 마련
    모듈식 구조를 통해 향후 지형, 식생 등 추가 변수의 단계적 통합 및 공공 데이터 API 연동을 위한 기술적 토대 확보.

Development Process

ver.01: Core Concept Validation

Problem: Text-based forecasts lacked spatial context.
Action: Built an MVP with a simple fan-chart to test the core hypothesis that map-based visuals would be more effective for field decisions.

Change: Text-based report → Map-based visual forecast.

Tech: Vanilla JS, Leaflet.js

핵심 가설 검증

문제: 텍스트 예보는 공간적 맥락이 부족.
해결: 지도 기반 시각화가 더 효과적일 것이라는 핵심 가설을 검증하기 위해, 간단한 부채꼴 차트 형태의 MVP를 구축.

ver.02: Enhancing Visual Realism

Problem: The initial fan-chart was too abstract for precise analysis.
Action: Replaced it with a grid-based heatmap using interpolation, significantly improving visual realism and optimizing the UX for mobile.

Change: Abstract fan-chart → Granular grid heatmap.

Tech: Client-side Interpolation, Responsive CSS

시각적 현실감 강화

문제: 초기 부채꼴 차트는 정밀한 분석에 한계.
해결: 시각적 현실감을 높이기 위해 보간법을 적용한 그리드 기반 히트맵으로 교체하고, 모바일 UX를 최적화.

ver.03: Upgrading the Prediction Engine

Problem: A static model could not adapt to changing weather.
Action: Upgraded the prediction engine with a dynamic weighting system, achieving a stable cosine similarity of 0.5875 with the RandomForest model.

Change: Static prediction model → Adaptive engine with dynamic weighting.

Tech: Python, Scikit-learn (RandomForest), SHAP

예측 엔진의 지능화

문제: 고정된 모델은 변화하는 기상에 적응 불가.
해결: RandomForest 모델로 안정적인 코사인 유사도 0.5875를 달성하는 동적 가중치 시스템을 도입하여, 상황 적응적 정확도를 향상.

ver.04: Ensuring Operational Reliability

Problem: Disjointed information and UI bugs were hindering quick decision-making.
Action: Overhauled the UI, integrating all data into a stable, single-view dashboard and eliminating 100% of critical layout bugs.

Change: Disjointed UI → Stable, single-view dashboard.

Tech: Flexbox, Advanced CSS, DOM Manipulation

운영 신뢰성 확보

문제: 분산된 정보와 UI 버그가 빠른 의사결정을 저해.
해결: 모든 데이터를 안정적인 단일 대시보드에 통합하는 UI 개편을 통해 치명적 레이아웃 버그를 100% 해결하고 운영 신뢰성을 확보.

ver.05: Providing Decision Confidence

Problem: Users needed to know not just the prediction, but how much to trust it.
Action: Introduced a visualized uncertainty score (±%) for each of the 8 directions, evolving the tool into a quantifiable decision-support instrument.

Change: Forecast-only output → Forecast + Quantified Uncertainty.

Tech: Client-side Dynamic Calculations

의사결정의 확신 제공

문제: 사용자는 예측뿐 아니라 그 신뢰도까지 파악해야 할 필요.
해결: 8개 방향별 불확실성(±%) 점수를 시각화하여, 예보 도구를 정량적 의사결정 지원 도구로 발전시킴.
